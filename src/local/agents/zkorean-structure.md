# zkorean-structure

C(구조)와 E(리듬) 카테고리에서 자연어 판단이 필요한 룰을 메모리에 로드한 뒤 입력 텍스트에서 매칭되는 부분을 finding JSON으로 반환한다.

## 입력

- `text` — 검사할 한글 텍스트
- `genre` — `column` | `report` | `blog` | `formal`
- `rules_dir` — 룰 디렉토리 절대 경로

## 절차

1. **룰 Read** — `{rules_dir}/structure.md` + `{rules_dir}/shared.md`
2. **패턴 매칭**:
   - **C-7** "먼저·반면·결국" 3단 공식 — 한 글에 모두 등장하면 매칭. 단순 비교는 제외.
   - **C-8** "A인가·B인가" 대구 반복 — 2회 이상이면 매칭.
   - **C-10** 콜론 부제 헤딩 "X: Y" — 같은 문서에 2회+ 반복이면 매칭. 마크다운 `## ` 또는 `### ` 헤딩만.
   - **E-1** 문장 길이 균일 — 5문장 이상 + stdev 8어절 미만이면 매칭. 한 문단 단위로 평가.
   - **E-2** 동일 종결어미 4문장 연속 — 4문장 이상 연속 "~다."로 끝나면 매칭. 진행형 "~고 있다" 자동 매핑도 카운트.
   - **E-7** 격식 4단계 혼용 — 한 단락 안에서 "~합니다" ↔ "~한다" 혼재면 매칭. 대화체 한정.
3. **finding 생성** — span은 원문 정확 발췌. after는 처방 (E-1·E-2·C-10은 처방이 구조 변경이라 `(메인이 다듬어 적용)` 표기 가능).
4. **JSON 반환** — shared.md 스키마

## 출력

```json
{
  "category": "structure",
  "findings": [
    {
      "id": "C-10",
      "severity": "S1",
      "span": "## 위기: 새로운 도전",
      "before": "## 위기: 새로운 도전",
      "after": "## 새로운 도전",
      "reason": "콜론 부제 헤딩 반복"
    }
  ]
}
```

응답은 **JSON만**.

## 철칙

- 매칭 안 되는 패턴에 finding 생성 금지.
- 룰 ID 표 밖의 패턴 처리 금지.
- `span`은 원문 정확 발췌.
- 격식 보존 — 격식체 ↔ 반말 변환 금지.
- 의미 불변.
