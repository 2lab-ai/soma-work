# Shared rules — Do-NOT, severity, self-check, grading

모든 zkorean 에이전트가 공통으로 따르는 규칙.

## Do-NOT — 탐지·교정 모두 제외

- 고유명사·제품명·모델명·기관명·인명
- 수치·날짜·단위
- 큰따옴표 안 직접 인용
- 법률 조문, 수학·화학·통계 표기
- 영어 약어 (LLM·GPU·MCP·API 등 업계 표준)

## 심각도

- **S1** — 한 번이라도 나오면 교정
- **S2** — 2회+ 누적 시 교정
- **S3** — 약한 신호. 다른 패턴과 중첩 시에만

## 과윤문 가드

- 변경률 30% 초과 → 경고
- 변경률 50% 초과 → 중단·원문 반환

## Finding JSON 스키마 (모든 서브 에이전트가 반환)

```json
{
  "category": "translationese|structure|modifiers",
  "findings": [
    {
      "id": "A-2",
      "severity": "S1",
      "span": "원문에서 그대로 발췌한 문자열",
      "before": "교정 전",
      "after": "교정 후",
      "reason": "한 줄 사유"
    }
  ]
}
```

- `span` — 원문에 정확히 등장하는 문자열. 메인이 적용 시 이 문자열을 찾아 `after`로 치환.
- 매칭 안 되는 패턴은 **finding 생성 금지**. 추측·창작 금지.
- 동일 span에 여러 finding이 있으면 **심각도 높은 것 우선**(S1 > S2 > S3).

## 자체검증 (메인 에이전트가 적용 후 실행)

1. 고유명사·수치·날짜·인용 100% 보존 — 원문과 한 글자도 다르지 않은가
2. 변경률 30% 이하 — 50% 초과면 중단하고 원문 반환
3. 장르 유지 — 칼럼이 에세이·문학으로 떨어지거나 리포트가 블로그체로 가지 않았는가
4. 격식 보존 — 격식체 ↔ 반말 변환 금지
5. 잔존 S1 패턴 0건 — D-1~D-7, A-7, A-8, A-16, C-5, C-10, C-11, H-1, I-1, J-2
6. 인공 표현 자제 — 원문에 없던 비유·수사·문학적 표현을 임의로 추가하지 않았는가

위반 항목이 있으면 해당 변경을 되돌린다.

## 등급 (자가 채점)

- **A** — S1 잔존 0, S2 잔존 ≤ 2, 변경률 10~25%, 자체검증 6항 통과
- **B** — S1 잔존 0, S2 잔존 ≤ 4, 자체검증 5항 이상 통과
- **C** — S1 잔존 1~2 또는 자체검증 4항 이하 — 잔존 finding 명시
- **D** — S1 잔존 3+ 또는 변경률 50% 초과 — 원문 반환 + 사유
