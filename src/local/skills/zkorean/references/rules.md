# zkorean rules — AI 한글 작문 패턴 가이드

AI(ChatGPT·Claude·Gemini 등)가 한글로 글을 쓸 때 자주 드러내는 표면 패턴과, 같은 패턴이 사람 글에서도 정상으로 쓰이는 경계(register caveat)를 함께 적은 가이드. 에이전트는 이 가이드를 읽고 **문장 수준에서 어색한지 자연어로 판단**한 뒤에만 교정한다.

원본 분류 체계는 [`epoko77-ai/im-not-ai`](https://github.com/epoko77-ai/im-not-ai) v2.0 (MIT)에서 가져왔다. 검증 라운드 1에서 broken으로 판정된 ID는 제거하거나 hint로 격하했다. 자세한 내역은 SKILL.md의 검증 결과 섹션.

## Do-NOT — 탐지·교정 모두 제외

- 고유명사·제품명·모델명·기관명·인명
- 수치·날짜·단위
- 큰따옴표 안 직접 인용
- 법률 조문, 수학·화학·통계 표기
- 영어 약어 (LLM·GPU·MCP·API 등 업계 표준)

## 룰 (자연어 판단 가이드)

각 ID는 "이 표면 패턴이 있을 때, 이 문장에서 실제로 어색한지"를 LLM이 판단한다. 카운트 임계값은 없다. 한 번이라도 명백히 어색하면 교정, 아니면 통과.

> **register caveat**가 붙은 룰은 신문 사설·법조문·학술·백과체에서 정상 사용된다. 같은 표면형이 나오더라도 register와 맥락을 보고 판단.

### A. 번역투

| ID | 패턴 | register caveat | 처방 |
|---|---|---|---|
| A-1 | "~에 대해(서)" | 학술·논평에서 정상. *X에 대해 무엇을 말하다*는 자연. | 직역으로 어색한 경우만 목적격 직결 ("X에 대해 논의" → "X를 논의") |
| A-2 | "~를 통해/통하여" 남발 | 단일 등장은 정상. | 한 단락에 3회+ 반복으로 어색하면 "~로", "~해서"로 분산 |
| A-3 | "~에 있어(서)" | 격식 학술체에서 정상. | 공허한 도입부일 때만 "~에서", "~을 볼 때"로 |
| A-9 | "~에 의해" 피동 | 행위자 부각이 필요한 경우 정상. | 행위자가 명백한데도 피동이면 능동으로 ("AI에 의해 생성" → "AI가 만든") |
| A-10 | "~할 수 있다" 남발 | 가능성·역량 표현은 정상. | 단정 회피로 남용된 경우만 단언으로 |
| A-11 | "~을 위해" 목적절 남발 | 단일 등장은 정상. | 같은 단락 3회+에서 "~려고", "~위한"으로 분산 |
| A-15 | 추상 주어 + 만능 동사 (보여준다·시사한다·만들어낸다) | 학술 인용에선 정상. | 보고서·기사체에서 "X는 보여준다" → "X에 따르면 ~이다" |
| A-16 | "그/그녀/그것/그들" 반복 | 1~2회는 정상. | 한 단락 3회+ 영어 직역식이면 50%+ 영형(생략) (김도훈 2009) |
| A-18 | 명사 앞 ≥ 4어절 좌향 수식 | 한국어 자체가 좌향 수식 언어라 3어절까지는 정상. | 4어절 이상이면 분리 또는 후치 동격절 (박옥수 2018) |

폐기된 ID (라운드 1 BROKEN): **A-7** (light verb "가지고 있다") — 위키·백과체 정상. **A-8/A-19** — 이중 피동/이중 조사 표면형이지만 자연어 판단으로 보강하지 않으면 false positive 위험.

## Severity

심각도 카운트는 더 이상 등급 산정에 쓰지 않는다. 글 전체가 자연스럽게 읽히는지가 기준.

각 finding을 2개 분류로:
- **edit** — 명백히 어색해서 교정해야 함
- **hint** — 문장 수준에서 자연스럽지만 누적 패턴이 보일 때 사용자에게 알리는 신호

```json
{

  "change_rate": 0.18,

  ],

    {"id": "H-3", "occurrences": 4, "note": "학술 register지만 한 번만 사용 권장"}
  ]
```

## 보존 원칙 (위반 시 즉시 롤백)

1. **의미 불변** — 사실·주장·수치·날짜·고유명사·인용 100% 일치
2. **격식 보존** — 격식체 ↔ 반말 변환 금지
3. **장르 유지** — 칼럼이 에세이로, 리포트가 블로그로 떨어지지 않게
4. **register 보존** — 같은 표면형이라도 학술/공적/사설 register에서 정상이면 건드리지 않는다
5. **과윤문 금지** — 변경률 50% 초과 시 중단·원문 반환
6. **Do-NOT 보존** — 영어 약어·고유명사·수치·인용 원형
